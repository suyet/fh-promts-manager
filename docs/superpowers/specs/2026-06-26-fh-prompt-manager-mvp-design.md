# FH Prompt Manager MVP 设计文档

日期：2026-06-26
状态：待用户审阅

## 文档约定

本项目后续产品设计、实施计划、技术说明和交付文档均默认使用中文编写。代码命名、文件名、接口字段名仍使用英文。

## 目标

FH Prompt Manager 是一个本地优先的 Chrome/Edge 浏览器插件，用于个人提示词资产管理。MVP 重点解决三个问题：

- 提示词资产散落在 Notion、本地文件、聊天记录等位置，难以查找。
- 高质量提示词在迭代过程中容易丢失，缺少清晰的版本记录。
- 敏感业务提示词不希望存放在第三方云端，需要本地保存和离线可用。

MVP 定位为个人本地知识库。不包含团队协作、云同步、权限体系、网页输入框自动插入，也不考虑上架浏览器插件商店。

## 产品形态

插件包含两个主要界面：

- Fast Use 弹窗：用于高频搜索、场景筛选、最近使用和复制。
- Manager 管理页：用于完整资产库管理、提示词详情编辑、版本历史、Diff、导入和导出。

插件通过 Manifest V3 实现，并通过 Chrome/Edge 开发者模式安装。

## 视觉方向

整体界面采用白色和浅灰色工作区，蓝色作为主色。

Manager 页面使用固定顶部导航栏。首页采用两栏资产库布局：

- 左栏：场景列表。
- 右栏：当前场景或筛选条件下的 Prompt 卡片。

Prompt 详情页也采用两栏布局：

- 左栏：提示词编辑器。
- 右栏：详细信息和版本历史。

## Fast Use 弹窗

弹窗用于快速查找和复制，不承载复杂编辑能力。

必需元素：

- 搜索输入框。
- 场景快速筛选。
- 显式的“最近使用”栏目。
- 匹配的 Prompt 结果列表。
- 复制某个 Prompt 最新版本的操作。
- 打开 Manager 管理页的入口。

MVP 不支持把 Prompt 自动插入网页输入框，只支持复制到剪贴板。

## Manager 首页

Manager 首页是提示词资产库。

必需元素：

- 固定顶部导航栏。
- 带图标和颜色的场景列表。
- Prompt 卡片。
- 搜索和筛选控件。
- 导入和导出入口。
- 新建场景和新建 Prompt 操作。

Prompt 卡片应展示足够用于快速浏览的信息：

- Prompt 标题。
- 所属场景。
- 标签。
- 最新版本号。
- 收藏状态。
- 最近使用时间。
- 复制操作。

## Prompt 详情页

进入 Prompt 详情页后，在固定顶部导航栏下方展示一个二级标题行。

二级标题行：

- 左侧：返回/退出操作、Prompt 名称、最新版本标识。
- 右侧：Prompt 级操作，包括收藏、删除、复制。

主内容区域：

- 左栏：提示词编辑器。
- 右栏：详细信息和版本历史。

编辑器只包含提示词正文。它应呈现为轻量代码编辑器风格，不混入标题、描述、场景、标签等元信息。

编辑器标题栏：

- 左侧：编辑器标签，例如“Prompt”。
- 右侧：保存、下载、复制。

编辑行为：

- 数据模型中没有草稿概念。
- 编辑器从最新已保存版本加载内容。
- 点击保存会创建一个新版本。
- 离开页面且未保存时，未保存的编辑内容会被丢弃。
- 新建 Prompt 时，只有用户保存后才写入数据；第一次保存创建 `v1`。
- 标题、场景、标签、描述、收藏状态等 Prompt 元信息可在详情页或右侧栏编辑，但编辑器正文区域只放提示词内容。

操作行为：

- 二级标题行的复制按钮复制最新已保存版本。
- 编辑器标题栏的复制按钮复制当前编辑器中可见内容，包括未保存编辑。
- 编辑器标题栏的下载按钮将当前编辑器中可见内容下载为本地文本或 Markdown 文件。

## 版本管理

每一次保存的提示词正文都会成为一个版本。

规则：

- Prompt 正文内容存放在 `PromptVersion` 中，不直接存放在 `Prompt` 上。
- 保存时创建新的 `PromptVersion`，版本号递增。
- 最新版本是弹窗复制、管理页复制和编辑器加载时的默认内容。
- 历史版本创建后不可变。
- 用户可以从版本历史中复制任意版本。

MVP 不包含静默自动保存。

## Diff

Diff 是 Prompt 详情页中的辅助工作流，不是顶层导航入口。

MVP 范围：

- 只支持“某个历史版本 vs 最新版本”。
- 不支持两个历史版本之间互相比较。
- 不比较未保存编辑内容。
- 不支持恢复旧版本。
- 不支持合并或逐块接受 Diff。

交互：

- 右侧版本历史展示最新版本和历史版本。
- 最新版本只显示复制操作。
- 每个历史版本显示“Compare to Latest”和复制操作。
- Diff 页面打开后，左侧固定为历史版本，右侧固定为最新版本。
- Diff 页面顶部有标题/操作行，右侧放置复制历史版本、复制最新版本、返回操作。

渲染：

- 使用按行 Diff。
- 删除或旧内容用红色高亮。
- 新增或最新内容用绿色高亮。
- 未变化内容使用中性样式。

## 数据模型

IndexedDB 是主数据库。

### Scene

字段：

- `id`
- `name`
- `description`
- `icon`
- `color`
- `sortOrder`
- `createdAt`
- `updatedAt`

说明：

- Scene 是主要业务分类。
- Scene 只支持单层结构。
- 图标来自内置图标集，不支持用户上传图片。
- 颜色来自预设色板。

### Prompt

字段：

- `id`
- `sceneId`
- `title`
- `description`
- `tags`
- `favorite`
- `latestVersionId`
- `latestVersionNumber`
- `lastUsedAt`
- `createdAt`
- `updatedAt`

说明：

- 一个 Prompt 只属于一个 Scene。
- 标签是用户自定义字符串。
- 标签用于展示、搜索和筛选。
- Prompt 正文内容不存放在 `Prompt` 上。

### PromptVersion

字段：

- `id`
- `promptId`
- `versionNumber`
- `content`
- `note`
- `createdAt`

说明：

- 版本创建后不可变。
- `note` 可选，用于记录该版本的变更说明。

### UsageRecord

字段：

- `id`
- `promptId`
- `versionId`
- `usedAt`
- `source`

说明：

- 用户复制 Prompt 时创建使用记录。
- 用于“最近使用”和最近使用时间展示。

## 导入与导出

导出：

- 将全部数据导出为一个 JSON 文件。
- 文件包含 `schemaVersion`、导出时间、Scenes、Prompts、Versions 和 UsageRecords。

导入：

- 导入前必须有预览步骤，不直接写入。
- 预览展示将新增或合并的 Scene、Prompt、Version 数量。
- JSON 无法解析时不写入任何数据。
- `schemaVersion` 不支持时不写入任何数据。

合并规则：

- 同名 Scene 合并。
- 如果导入的 Scene 图标或颜色与本地不同，保留本地设置。
- 同一 Scene 下同名 Prompt 视为冲突。
- 冲突 Prompt 合并版本。
- 如果导入的最新内容本地不存在，则作为一个版本保留下来。
- 导入不会静默删除或覆盖本地数据。

MVP 不包含全量替换式导入。

## 技术架构

推荐技术栈：

- Manifest V3 浏览器插件。
- React + TypeScript + Vite。
- 使用 Dexie 或类似轻量封装访问 IndexedDB。
- 使用 `lucide-react` 作为图标库。
- 使用本地 service 层封装数据库操作。
- 使用成熟 Diff 库生成按行 Diff。

插件组成：

- `popup`：Fast Use 弹窗。
- `manager`：完整管理页。
- 共享 IndexedDB 数据服务。
- 共享导入/导出服务。
- 共享 Diff 工具。

权限保持最小化：

- IndexedDB 不需要额外插件权限。
- 剪贴板复制使用插件 UI 环境下的浏览器剪贴板 API。
- 不使用 content script 权限。
- 不申请 host permissions。
- 不使用远程服务器。

## 错误处理

必需场景：

- IndexedDB 不可用：展示明确的本地存储不可用提示。
- 数据库写入失败：保留当前表单内存状态，并提示用户重试。
- 剪贴板复制失败：展示 Prompt 内容，允许用户手动复制。
- 导入 JSON 解析失败：提示文件格式错误，不写入任何数据。
- 导入 schema 不匹配：提示版本不支持，不写入任何数据。
- 删除 Prompt：需要确认，因为该 Prompt 下的版本会一并删除。
- 离开详情页且存在未保存编辑：提示未保存内容将被丢弃。

## MVP 不包含

以下内容不进入 MVP：

- 团队协作。
- 云同步。
- 用户账号。
- 浏览器插件商店上架。
- 网页输入框自动插入。
- Content scripts。
- 快捷键。
- Prompt 变量或填写表单。
- 多级分类。
- 用户上传图标或图片。
- 自动保存未保存编辑。
- 历史版本之间互相 Diff。
- 恢复旧版本。
- Diff 合并工具。

## 验收标准

MVP 完成时应满足：

- 用户可以创建、编辑、删除、排序 Scene。
- Scene 支持内置图标和预设颜色。
- 用户可以创建、编辑、删除、收藏 Prompt，并为 Prompt 添加标签。
- 保存 Prompt 内容会创建一个新版本。
- 详情页存在未保存编辑时，离开前会提示；确认离开后未保存内容被丢弃。
- 用户可以从弹窗和管理页复制最新版本。
- 用户可以从版本历史复制旧版本。
- 弹窗支持搜索、场景快速筛选和最近使用。
- Manager 首页展示场景列表和 Prompt 卡片。
- Prompt 详情页使用已确认的二级标题行和编辑器布局。
- Diff 可以比较某个历史版本和最新版本。
- 导出能生成有效 JSON 备份。
- 导入能预览变更并执行合并导入。
- 刷新浏览器或重新打开插件后，数据仍然存在。
- 核心功能在离线状态下可用。
